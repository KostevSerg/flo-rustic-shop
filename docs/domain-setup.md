# Инструкция по подключению домена к poehali.dev

## Шаг 1: Подготовка домена

Убедитесь, что у вас есть:
- Зарегистрированный домен (например, `florustic.ru`)
- Доступ к панели управления DNS-записями у вашего регистратора (Reg.ru, Timeweb, Cloudflare и т.д.)

## Шаг 2: Подключение домена в poehali.dev

1. Откройте ваш проект в редакторе poehali.dev
2. Нажмите кнопку **"Опубликовать"** в верхнем меню
3. Выберите **"Привязать свой домен"**
4. Введите ваш домен (например, `florustic.ru`)
5. Система покажет DNS-записи, которые нужно добавить

## Шаг 3: Создание DNS-записей

### Вариант 1: Корневой домен (florustic.ru)

Добавьте следующие записи в панели управления DNS:

#### A-запись (основная)
```
Тип:     A
Имя:     @ (или оставьте пустым)
Значение: IP-адрес из poehali.dev
TTL:     3600 (или 1 час)
```

#### CNAME-запись для www
```
Тип:     CNAME
Имя:     www
Значение: ваш-проект.poehali.dev
TTL:     3600
```

### Вариант 2: Поддомен (shop.florustic.ru)

Добавьте только CNAME-запись:

```
Тип:     CNAME
Имя:     shop (или другое имя поддомена)
Значение: ваш-проект.poehali.dev
TTL:     3600
```

## Шаг 4: Инструкции для популярных регистраторов

### Reg.ru

1. Войдите в личный кабинет на reg.ru
2. Перейдите в раздел **"Домены"** → выберите ваш домен
3. Нажмите **"Управление зоной"** или **"DNS-серверы и зона"**
4. Добавьте записи:
   - Нажмите **"Добавить запись"**
   - Выберите тип (A или CNAME)
   - Заполните поля согласно данным из poehali.dev
   - Нажмите **"Добавить"**
5. Сохраните изменения

### Timeweb

1. Войдите в панель управления Timeweb
2. Перейдите в **"Домены"** → **"DNS-записи"**
3. Выберите ваш домен
4. Нажмите **"Добавить запись"**
5. Заполните данные и сохраните

### Cloudflare

1. Войдите в панель Cloudflare
2. Выберите ваш домен
3. Перейдите в раздел **"DNS"**
4. Нажмите **"Add record"**
5. Заполните поля:
   - Type: A или CNAME
   - Name: @ или www
   - Content: значение из poehali.dev
   - Proxy status: можно оставить включенным (оранжевое облако)
6. Нажмите **"Save"**

### Nic.ru

1. Войдите в личный кабинет Nic.ru
2. Перейдите в **"Услуги"** → **"Домены"**
3. Выберите домен → **"DNS-серверы"**
4. Убедитесь, что используются DNS-серверы Nic.ru
5. Перейдите в **"Управление зоной"**
6. Добавьте необходимые записи
7. Сохраните изменения

## Шаг 5: Проверка настройки DNS

После добавления записей подождите 10-60 минут (время распространения DNS).

Проверить правильность настройки можно через:

### Командная строка (Windows)
```bash
nslookup florustic.ru
nslookup www.florustic.ru
```

### Командная строка (Mac/Linux)
```bash
dig florustic.ru
dig www.florustic.ru
```

### Онлайн-сервисы
- https://www.whatsmydns.net/
- https://dnschecker.org/

Убедитесь, что DNS-записи возвращают правильные значения из poehali.dev.

## Шаг 6: Активация SSL-сертификата

После распространения DNS-записей:

1. Вернитесь в редактор poehali.dev
2. В разделе **"Опубликовать"** → **"Домены"** нажмите **"Проверить"**
3. Система автоматически выпустит SSL-сертификат (Let's Encrypt)
4. Через 1-5 минут ваш сайт будет доступен по HTTPS

## Часто задаваемые вопросы

### Сколько времени занимает подключение домена?
- Добавление DNS-записей: 2-5 минут
- Распространение DNS: 10-60 минут (иногда до 24 часов)
- Выпуск SSL-сертификата: 1-5 минут

### Что делать, если домен не работает через 24 часа?
1. Проверьте правильность DNS-записей через онлайн-сервисы
2. Убедитесь, что используете правильные DNS-серверы
3. Очистите кеш DNS на компьютере:
   - Windows: `ipconfig /flushdns`
   - Mac: `sudo dscacheutil -flushcache`
   - Linux: `sudo systemd-resolve --flush-caches`
4. Обратитесь в поддержку poehali.dev: https://t.me/+QgiLIa1gFRY4Y2Iy

### Можно ли использовать несколько доменов для одного проекта?
Да, можно добавить несколько доменов через раздел "Опубликовать" → "Домены".

### Нужно ли переносить хостинг?
Нет! При подключении домена к poehali.dev ваш проект автоматически хостится на серверах Яндекс.Облака. Переносить ничего не требуется.

### Можно ли использовать Cloudflare с poehali.dev?
Да, Cloudflare отлично работает с poehali.dev. Можете включить Proxy (оранжевое облако) для дополнительной защиты и кеширования.

## Примеры типичных ошибок

### Ошибка 1: "DNS_PROBE_FINISHED_NXDOMAIN"
**Причина:** DNS-записи не найдены или неправильно настроены  
**Решение:** Проверьте правильность добавленных DNS-записей

### Ошибка 2: "Сертификат недействителен"
**Причина:** SSL-сертификат еще не выпущен  
**Решение:** Подождите 5-10 минут, затем обновите страницу

### Ошибка 3: "ERR_CONNECTION_REFUSED"
**Причина:** DNS-записи указывают на неправильный адрес  
**Решение:** Перепроверьте значения в DNS-записях из poehali.dev
